# OpenAPI / Swagger Reference

The OmniExtract Gateway API is fully documented via Swagger UI and ReDoc, automatically generated by FastAPI.

## Interactive Documentation

| Interface | URL |
| :--- | :--- |
| Swagger UI | `http://localhost:8000/docs` |
| ReDoc | `http://localhost:8000/redoc` |
| OpenAPI JSON | `http://localhost:8000/openapi.json` |

## Exporting the OpenAPI Schema

```bash
# Export to JSON
curl http://localhost:8000/openapi.json > docs/openapi.json

# Export to YAML (requires yq)
curl http://localhost:8000/openapi.json | yq -P > docs/openapi.yaml
```

## Key Endpoints

### `POST /extract`

Route a document extraction request to any supported engine.

**Request body:** `ExtractionRequest`  
**Response:** `StandardizedExtraction`

### `GET /prebuilts`

List all available prebuilt document templates.

### `POST /prebuilts/custom`

Register a new custom prebuilt template. The gateway auto-generates the optimal LLM extraction prompt from the provided field list.

### `GET /registry/models`

Return the full model registry grouped by provider.

### `GET /health`

Health check endpoint.
